## 📝 文档维护说明

### 更新频率
- **更新日志.md**: 每个版本发布时更新（必须）
- **开发路线图.md**: 每月审查并调整
- **快速启动指南.md**: 功能变化时更新
- **系统说明文档**: 对应功能开发时创建/更新
- **其他文档**: 根据需要更新

### 文档规范
- ✅ 使用Markdown格式
- ✅ 保持目录结构清晰
- ✅ 添加适当的emoji提升可读性
- ✅ 及时归档过时文档到 `archive/`
- ✅ 重大功能完成后合并临时文档到主文档

### 归档策略
- 完成总结类文档 → 合并到更新日志后归档
- 规划类文档 → 合并到开发路线图后归档
- 过时的设计文档 → 直接归档
- 临时文档 → 删除或归档
