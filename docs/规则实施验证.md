## 🎯 实施的三大规则

### 1️⃣ 所有文档必须在 docs/
```
✅ 根目录: README.md (唯一例外)
✅ 永久文档: docs/*.md
✅ 测试文档: docs/测试/*.md
❌ 不要在根目录创建其他 MD
❌ 不要在 src/ 创建文档
```

### 2️⃣ 必须更新旧代码和旧文档
```
✅ 代码变更 → 更新文档
✅ 新功能 → 更新已完成总结、优化计划、更新日志
✅ 检查旧代码 → 更新而非新建
❌ 不要创建重复文件
❌ 不要让文档过期
```

### 3️⃣ 功能完成后删除测试文档
```
✅ 功能开发 → 创建测试文档 (docs/测试/)
✅ 功能完成 → 更新主文档
✅ 测试通过 → 删除测试文档
❌ 不要保留已完成功能的测试文档
```
